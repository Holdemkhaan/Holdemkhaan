<summary><h2>💬 WhatsApp Translator</h2></summary>

**Бот-переводчик для групповых чатов WhatsApp (Русский ↔ Индонезийский)**

### Статус: ✅ 100% — Работает на VPS, проходит обкатку

### Что делает бот
- Слушает все сообщения в групповых чатах WhatsApp
- Определяет язык (русский / индонезийский / другой)
- Переводит: RU → ID, ID → RU, другие → на ОБА языка
- Отвечает в чат: `🔄 Имя отправителя: перевод`

### Стек технологий

| Компонент | Технология |
|-----------|------------|
| Runtime | Node.js v20 |
| AI перевод | Claude Sonnet API |
| WhatsApp клиент | whatsapp-web.js (Puppeteer/Chrome) |
| Процесс-менеджер | PM2 (автозапуск при перезагрузке) |
| Сервер | VPS (Linux) |

### Проделанная работа

#### Этап 1: Baileys (отброшен)
| Что делали | Результат |
|------------|-----------|
| Написали бота на библиотеке Baileys | ❌ WhatsApp блокирует Baileys на VPS (HTTP 405) |
| Добавили анти-бан систему (rate limiter, очередь, задержки) | ❌ Не помогло — блокировка на уровне IP |
| Подключили прокси через https-proxy-agent | ❌ Не решило проблему |
| Написали 3 скрипта для привязки (QR, pairing code, direct) | ⚠️ Работали, но бот не держал соединение |

#### Этап 2: whatsapp-web.js (рабочее решение)
| Что делали | Результат |
|------------|-----------|
| Переписали бота на whatsapp-web.js (реальный Chrome) | ✅ WhatsApp не блокирует |
| Убрали прокси (Chrome проходит проверки сам) | ✅ Проще и стабильнее |
| Убрали искусственные задержки (не защищают от банов) | ✅ Быстрее работает |
| Написали скрипт привязки через QR-код в PNG | ✅ Работает через SCP |

#### Этап 3: Надёжность и тесты
| Что делали | Результат |
|------------|-----------|
| Вынесли всю логику в библиотеку `lib.mjs` | ✅ Чистая архитектура |
| Написали **53 юнит-теста** | ✅ Все проходят |
| Добавили мониторинг памяти (рестарт при >800MB) | ✅ Защита от утечек Chrome |
| Catchup: догоняет пропущенные сообщения (до 30 мин) | ✅ Ничего не теряется |
| Дедупликация сообщений (TTL 1 час, лимит 1000) | ✅ Нет дублей |
| Graceful shutdown (сохраняет timestamp при остановке) | ✅ Плавный рестарт |
| Автопереподключение при разрыве связи | ✅ 10 сек задержка |
| Ретраи Claude API (3 попытки) | ✅ Устойчивость к ошибкам |

### Архитектурные решения

| Решение | Почему |
|---------|--------|
| whatsapp-web.js вместо Baileys | Baileys блокируется на серверных IP |
| Без прокси | Chrome проходит проверки WhatsApp сам |
| Без искусственных задержек | Баны за массовую рассылку, не за скорость ответа в группе |
| Мониторинг памяти 800MB | Puppeteer/Chrome течёт по памяти |
| Вся логика в lib.mjs | Удобно тестировать отдельно от WhatsApp |
